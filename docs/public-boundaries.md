# Public boundaries

This page explains what can be shown publicly and what stays private.

The aim is to share the project clearly without exposing private implementation details, sensitive data, or material that could make the working system less safe.

## Public

The public repository can include:

- high-level progress updates
- selected hardware photos
- safe UI screenshots without private data
- broad technical direction
- simplified example code
- public demo notes
- edited progress photos
- high-level diagrams
- short demo clips after review

## Private

The private project stays private. Do not publish:

- main source code
- exact internal architecture
- model routing details
- memory logic
- prompts
- diagnostics
- logs
- user data
- raw media
- private configs
- `.env` files
- secrets, keys, or tokens
- roadmap details

## Practical rule

If a file shows how the private system works internally, contains private data, or could expose credentials or personal information, it should not be added to this repository.

Public material should explain what the project is, what is being explored, and what has been learned at a safe high level.
