# Current work

This page explains what I am working on now.

## Voice interaction

I am working on local voice interaction so the assistant can respond to spoken input in a practical way.

This matters because voice is often the most natural way to use an assistant, especially when the system is connected to physical hardware or a screen.

The benefit could be a smoother assistant experience where simple requests can be handled without needing to type.

## Visual interface

I am working on a visual interface that can show feedback, status, and assistant responses.

This matters because a physical assistant should not only speak. It should also show what it is doing, what state it is in, and when it needs attention.

The benefit could be a clearer and more trustworthy interaction, because the user can see useful feedback instead of guessing what is happening.

## Local AI direction

The project is exploring how much assistant behaviour can happen locally, close to the user.

This matters because local processing can reduce dependence on cloud services and may give the user more control over private information.

The benefit could be a more private and reliable assistant for everyday tasks, where suitable work happens on local hardware.

## Hardware and sensors

I am working with Raspberry Pi hardware, displays, microphones, sensors, and related components.

This matters because the project is not only a software idea. It is being tested as a real physical system.

The benefit could be an assistant that understands more about its own hardware state and surrounding environment in a controlled way.

## Robotics and movement safety

The project is exploring robotics and movement with safety in mind.

This matters because anything that can move must be treated carefully. The system should avoid unsafe actions and make movement predictable.

The benefit could be a physical assistant that can eventually interact with its environment in useful but limited and controlled ways.

## Learning support

I am exploring how the assistant could support learning and understanding information.

This matters because a local assistant could help explain, organise, and review information in a more personal way.

The benefit could be better support for study, revision, and practical learning tasks, without claiming to replace teachers or proper guidance.

## Personal context and assistance concepts

The project is exploring ideas around reminders, routines, personal context, and everyday assistance.

This matters because useful assistants need some context, but that context must be handled carefully.

The benefit could be support for everyday organisation while keeping strong boundaries around what is stored and what is shared publicly.

## Testing and reliability

I am testing, debugging, and improving reliability as the software, hardware, voice, and interface parts come together.

This matters because a system connected to real hardware needs predictable behaviour, not only a good-looking demo.

The benefit could be a more stable foundation for future public demos, progress updates, and safe experiments.
