# What can be shown publicly

Use this checklist before adding material to the public repository or sharing it on LinkedIn.

## Safe to share after review

- hardware photos
- UI screenshots without private data
- edited progress photos
- short demo clips
- simplified examples
- high-level diagrams
- public progress notes
- broad technical summaries

## Do not share

- private code
- prompts
- memory
- logs
- diagnostics
- `.env` files
- tokens
- private configs
- raw recordings
- raw camera captures
- raw transcripts
- user data
- secrets or credentials

## Before publishing

Check the screen, background, terminal windows, filenames, browser tabs, and any visible text. Remove anything private before uploading an image, video, or document.
